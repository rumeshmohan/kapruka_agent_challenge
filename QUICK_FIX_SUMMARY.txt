PROBLEM:
--------
App deployed to Railway but crashed when users clicked buttons or 
triggered Streamlit events (Add to Cart, Remove, Clear Cart, etc.)

ROOT CAUSES:
-----------
1. Multiple st.rerun() calls causing infinite loops in production
2. Missing error handling in query processing
3. Improper PORT configuration for Railway
4. Removed unnecessary final st.rerun() after chat messages

--------------
✓ ui/app.py         - Fixed all st.rerun() calls + error handling

✓ .streamlit/config.toml - Production-ready configuration

TROUBLESHOOTING:
---------------

Error: "ValueError: API key not found"
Fix: Add missing API key in Railway Variables tab

Error: "ModuleNotFoundError"
Fix: Check requirements.txt, redeploy

Error: Out of Memory
Fix: Upgrade Railway plan or optimize code

Error: Port binding failed
Fix: Railway handles this automatically, just restart
